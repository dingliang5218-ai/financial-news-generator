## [1.0.0] - 2026-03-05

### 新增

#### 核心功能
- 多源 RSS 数据采集系统
- Claude AI 智能新闻分析
- 自动化内容生成（三种样式：简报、综合、深度）
- 定时任务调度系统
- 完整的错误处理和重试机制
- 系统健康检查（启动和运行时）
- SQLite 数据库存储
- 集中式日志管理

#### 调度任务
- 每日新闻生成（8:00 AM）
- 市场更新（工作日 9-17 点每小时）
- 周报总结（周日 18:00）
- 健康检查（每 6 小时）

#### 部署支持
- Docker 容器化部署
- Docker Compose 编排
- systemd 服务配置
- 快速启动脚本

- API 参考文档（API_REFERENCE.md）

#### 测试
- 集成测试套件
- 配置测试
- 日志测试
- 存储测试
- 错误处理测试
- 健康检查测试

### 修复

- 修复磁盘空间检查逻辑错误（启动时应该失败而不是警告）
- 修复日志文件命名格式（使用 YYYYMMDD 格式）
- 修复 Config 缺少 VERSION 属性
- 修复 Storage 缺少查询方法（get_article, search_articles 等）
- 修复 handle_errors 装饰器参数不兼容问题
- 修复测试用例中的断言错误

### 优化

- 优化错误处理装饰器，支持重试和错误类型配置
- 优化存储层，添加更多查询方法
- 优化日志系统，支持多个日志文件
- 优化数据采集，添加去重机制
- 优化健康检查，添加更多检查项

### 技术栈

- Python 3.8+
- Claude API (Anthropic)
- SQLite
- APScheduler
- feedparser
- python-dotenv

## 版本说明

### 版本号格式

版本号格式：`主版本号.次版本号.修订号`

- **主版本号**：不兼容的 API 修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正

### 变更类型

- **新增 (Added)**：新功能
- **变更 (Changed)**：现有功能的变更
- **弃用 (Deprecated)**：即将移除的功能
- **移除 (Removed)**：已移除的功能
- **修复 (Fixed)**：错误修复
- **安全 (Security)**：安全相关的修复
